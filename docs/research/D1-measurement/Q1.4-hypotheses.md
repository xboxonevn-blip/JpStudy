# D1.Q1.4 Hypotheses - Real Export Funnel Baseline

## H1.4.1 - Real GA4 export is the next hard blocker

Prediction: codebase-local tooling is ready enough to score real NS and SM1, but cannot answer "biggest leak today" without Analytics BigQuery export access or a downloaded event sample. Effect size: no real event counts can be produced from local files/CLI alone.

Falsifier: local Firebase/GCP tooling can query `analytics_*` tables and produce aggregate counts.

## H1.4.2 - A narrow event export is enough for first beta readout

Prediction: the 10-event allowlist in `tool/research/ga4_ns_export.sql` is sufficient to score NS and SM1 without exporting PII. Effect size: output can be consumed by both `north_star_report.dart --ga4-events` and `funnel_report.dart --ga4-events`.

Falsifier: report tools require user attributes or raw content values outside the event allowlist.

## H1.4.3 - Old data cannot reveal the biggest leak

Prediction: pre-Phase-0 exports, if present, will miss `onboarding_completed`, `n5_micro_quiz_completed`, or `session_quality_rated`, making leak analysis incomplete. Effect size: at least one SM1/NS stage count remains zero or missing.

Falsifier: current export already contains all required event families with stable user ids.
