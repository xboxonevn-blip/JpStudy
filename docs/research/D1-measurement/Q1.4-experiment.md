# D1.Q1.4 Experiment - Real Export Funnel Baseline

## Experiment E1.9

Question: What command/query is needed to answer Q1.4 with real admin data and no PII export?

Method:

- Use the official GA4 BigQuery schema: daily `events_YYYYMMDD` tables, `event_name`, microsecond UTC `event_timestamp`, `event_params`, `user_id`, and `user_pseudo_id`.
- Create a narrow BigQuery SQL file that exports only the event families needed by NS and SM1.
- Add a local BigQuery runner that uses `google-cloud-bigquery` and the `GOOGLE_APPLICATION_CREDENTIALS` environment variable.
- Document the run commands for `north_star_report.dart` and `funnel_report.dart`.

Expected information gain: medium. This does not produce real counts, but removes ambiguity from the handoff once BigQuery access exists.

SQL:

```text
tool/research/ga4_ns_export.sql
```

Runbook:

```text
python tool/research/bigquery_runner.py --sql tool/research/ga4_ns_export.sql --out docs/research/secure/ga4-ns-events.json
dart run tool/research/north_star_report.dart --ga4-events docs/research/secure/ga4-ns-events.json --window-start 2026-05-01T00:00:00.000Z
dart run tool/research/funnel_report.dart --ga4-events docs/research/secure/ga4-ns-events.json --window-start 2026-05-01T00:00:00.000Z
```

Before running, replace:

- `YYYYMMDD_START`
- `YYYYMMDD_END`

Current dataset target is pinned to `jpstudy-v2.analytics_536663906`.
