# D1.Q1.1 Observability Gap Map

Commit inspected: `8387e4440ed26aaea5caf1d51abb6023d234df18`

Question: What signals in Drift DB, Firebase Analytics, and SharedPreferences correlate with "user is learning"?

## Durable Drift Signals

| Surface | Evidence | Learning correlation | Gap |
| --- | --- | --- | --- |
| `user_progress` | `lib/data/db/tables.dart` tracks `streak`, `reviewedCount`, review grades, `xpEarned`, `lastStudyDate` | Daily engagement and review volume; maps to NS review gate locally | Local only; not enough for 50-user beta unless exported/joined |
| SRS tables | `srs_state`, `kanji_srs_state`, `grammar_srs_state`, `kana_srs_state` track stability/difficulty/last/next review | Retention model state and due backlog | Split by skill; no single NS cohort query yet |
| learn/test sessions | `lib/data/db/study_tables.dart` stores `correctCount`, `wrongCount`, `score`, `completedAt`, `xpEarned` | Quiz accuracy and session completion | N5 micro-quiz identity is a proxy today; lesson 1-25 tests may overcount |
| mistakes | `lib/data/db/mistake_tables.dart` tracks `wrongCount`, `lastMistakeAt`, source/prompt/answer | Weakness and remediation signal | Not directly part of NS; useful SM3/SM8 context |

## Firebase Analytics Signals

| Event | Evidence | Learning correlation | Gap |
| --- | --- | --- | --- |
| `study_session_start` / `study_session_complete` | `lib/core/analytics/analytics_service.dart`, learn summary logging | Funnel, session volume, accuracy proxy | Too coarse for NS by itself |
| `srs_review_completed` | `LessonRepository.saveTermReview` | Direct NS review count event | Needs real export sample and user continuity check |
| `n5_micro_quiz_completed` | `TestHistoryService.saveTest` for lesson ids 1-25 | NS quiz retention proxy | Needs explicit embedded micro-quiz id |
| `session_quality_rated` | learn/test summary star ratings | Direct NS quality gate | Analytics consent can suppress it; not persisted in Drift |
| auth/sync events | `auth_sign_in`, `cloud_upload`, `cloud_download` | Cohort/session context | Not learning evidence alone |

## SharedPreferences Signals

| Key family | Evidence | Learning correlation | Gap |
| --- | --- | --- | --- |
| onboarding | `prefOnboardingCompleted`, level, goal in `lib/core/onboarding_provider.dart` | SM1 funnel and JLPT segment | Device-local; not durable beta telemetry |
| daily session | `daily_session_progress_provider.dart` started/doneSteps/lastRoute/updatedAt | Daily critical path progress | Ad hoc local state; no export path |
| session drafts | `session_storage.dart` learn/test session snapshots | In-progress learning intent | Drafts can expire/delete; not canonical completion data |
| challenge/history | `challenge.history`, weekly challenge prefs | Engagement and repeated action | Gamification proxy, not NS proof |
| immersion/JLPT/study hub | `immersion_service.dart`, `jlpt_coach_service.dart`, `study_hub_provider.dart` | Advanced learner activity, reading/quiz snapshots | Fragmented feature state; no unified measurement contract |

## Gap Severity

P0:

- Real-user NS cannot be claimed until a real GA4/Firebase export sample scores through `tool/research/north_star_report.dart --ga4-events`.
- Stable beta denominator is missing: observed users can be counted, but recruited cohort membership and analytics-consent eligibility are not represented in the scorer.
- N5 micro-quiz is still a proxy based on lesson ids 1-25; an explicit quiz/event id is needed before interpreting retention.

P1:

- Session quality is event-only. If analytics is disabled or export fails, the quality gate disappears.
- Drift has rich local evidence, but there is no 1-command local DB cohort export yet.
- SharedPreferences contains useful funnel and daily-session hints but is not a reliable learning record.

## Current Answer

The strongest "user is learning" signals are SRS review completion, quiz accuracy, and session quality rating. Drift can observe review and quiz fragments locally; Firebase can now observe all three NS event families if analytics consent and export are available; SharedPreferences is useful for funnel/session context but is weak evidence because it is local, fragmented, and not cohort-safe.
