# D1.Q1.2 Analysis - Synthetic User Replay

## E1.6 Result

The report can now score 10 deterministic persona-tagged simulated users from one command. The simulator emits the same three event families used by export scoring: `srs_review_completed`, `n5_micro_quiz_completed`, and `session_quality_rated`.

## Hypothesis Update

- H1.2.1: supported for event-level replay. The CLI scores 10 users without local DB setup.
- H1.2.2: supported. The simulator cycles through N5, N4, N3, N2, and N1 persona tags and tests assert those tags survive event generation.
- H1.2.3: supported for regression use. Tests assert two runs with the same seed produce equivalent events.

## Red Team

Steelman opposite: this is not a real user session replay on top of Drift. It validates the NS event contract and personas, not app navigation, local DB writes, or actual FSRS state transitions. A stronger falsifier would run the Flutter app against a seeded local DB, perform scripted review/test/quality flows, export events, and score them without hand-built event generation.

## Action

Keep simulator as the Phase 0 regression harness. Next step is either a real GA4 export sample or a deeper app-level replay harness that drives Drift writes and analytics logging from the same seeded session.
