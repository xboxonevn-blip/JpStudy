# D1.Q1.3 Hypotheses - Minimum Analytics Event Set

Commit inspected: `8387e4440ed26aaea5caf1d51abb6023d234df18`

## H1.3.1 - The NS events are necessary but not sufficient for SM1-SM8

Prediction: `srs_review_completed`, `n5_micro_quiz_completed`, and `session_quality_rated` can compute NS gates but cannot compute onboarding funnel completion or per-level retention alone. Effect size: at least one additional event family is required.

Falsifier: existing events alone identify app open, onboarding completion, first SRS, and per-level day-N retention.

## H1.3.2 - Onboarding completion is the cheapest SM1 gap to close

Prediction: one `onboarding_completed` event with `level` and `goal` parameters materially improves SM1 with minimal product risk. Effect size: SM1 can distinguish onboarding done from first SRS.

Falsifier: onboarding completion is already observable in GA4 export or local durable analytics.

## H1.3.3 - Some sub-metrics should remain build/audit artifacts, not user events

Prediction: SM5 performance, SM6 accessibility, and SM7 editorial coverage should be measured by CI/audit reports rather than Firebase user telemetry. Effect size: event set remains smaller and less privacy-sensitive.

Falsifier: these sub-metrics require per-user runtime events to roll up into NS.
