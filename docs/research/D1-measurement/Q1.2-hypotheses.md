# D1.Q1.2 Hypotheses - Synthetic User Replay

Commit inspected: `8387e4440ed26aaea5caf1d51abb6023d234df18`

## H1.2.1 - Event-level replay is cheaper than DB mutation

Prediction: a deterministic simulator that emits the same NS telemetry events can validate the scoring path faster than mutating a local Drift DB. Effect size: one command scores 10 simulated users in under 5 seconds.

Falsifier: the CLI cannot score simulated event users without manual fixture editing or database setup.

## H1.2.2 - Persona tags expose segment mistakes early

Prediction: tagging simulated users with `persona` and `jlpt_level` will reveal whether the scorer preserves N5-N1 segmentation before real beta export exists. Effect size: a 10-user run contains all five levels N5-N1.

Falsifier: generated telemetry collapses segment identity or cannot round-trip through `NorthStarEventMapper`.

## H1.2.3 - Determinism matters more than behavioral realism in Phase 0

Prediction: fixed seed replay is enough for harness regression checks even though learner behavior is simplified. Effect size: two simulator runs with the same seed produce byte-equivalent event rows.

Falsifier: repeated runs produce different event order, user ids, or gate counts.
