# D8.Q8.5 Experiment - CI/CD Gates

## Experiment E8.7

Question: What launch gates are currently automated in GitHub Actions, and what remains manual?

Method:

- List `.github/workflows`.
- Read each workflow file.
- Read `package.json` scripts for Firebase rules tests.
- Compare observed CI gates to D8.Q8.5 requirements: analyze + test on every PR, smoke E2E on live URL after deploy, Lighthouse on deploy, and failure notification.
- Compare CI web build command to `SHIPPING.md` release build command.

Commands:

```text
Get-ChildItem .github\workflows -Force
Get-Content .github\workflows\ui-string-guard.yml
Get-Content package.json
rg "flutter-action|flutter analyze|flutter test|lighthouse|firebase deploy|test:storage|storage-rules|workflow" .github docs package.json pubspec.yaml -n
```

Expected information gain: high. CI coverage is inspectable without running remote GitHub Actions.

Limitations:

- This local audit cannot prove GitHub branch protection or notification settings.
- This does not run GitHub-hosted CI; it only verifies workflow definitions.
