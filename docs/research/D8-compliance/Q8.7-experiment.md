# D8.Q8.7 Experiment - Telemetry Retention and GDPR

## Method

1. Read consent, Analytics, legal, and data deletion code paths.
2. Grep for Analytics identity/deletion APIs and free-text telemetry risk.
3. Check official Google/Firebase docs for retention and deletion controls.
4. Attempt BigQuery dataset retention metadata query using the existing research runner.

## Result

- `analyticsConsentProvider` stores `analytics.consent`, defaults to disabled, hides the banner when Do Not Track is enabled, and calls `FirebaseAnalytics.instance.setAnalyticsCollectionEnabled(state.isGranted && !state.doNotTrack)`.
- `analyticsServiceProvider` only logs when consent is granted and Do Not Track is false.
- `AnalyticsService` logs coarse product events: session mode/counts, sign-in provider, onboarding level/goal, cloud sync trigger, SRS rating/level/interval, micro-quiz counts, and session quality rating.
- No `setUserId`, `setConsent`, `resetAnalyticsData`, or Analytics deletion UI was found.
- Data controls can delete Firebase Storage cloud backup, but not GA4 Analytics server-side records.
- Privacy copy says analytics should not include prompts, answers, names, or free-text learner content, but it remains a draft and does not specify retention duration.
- BigQuery retention metadata query could not run in the current shell because `google-cloud-bigquery` is not installed for `python`.

## Official Constraints

- GA4 standard retention choices are `2 months` or `14 months` for user-level data and key events; standard aggregated reports are not affected by the retention setting.
- Firebase Analytics collection can be toggled with `setAnalyticsCollectionEnabled`, and SDK APIs include `resetAnalyticsData` to clear local Analytics data and reset the app instance ID.
- GA Admin API `properties.submitUserDeletion` can submit deletion by `userId`, `clientId`, `appInstanceId`, or normalized user-provided data and requires `analytics.edit`.
- Firebase BigQuery export creates Analytics datasets/tables in BigQuery; retention/expiration is a BigQuery dataset/table policy question after export lands.
