# D8.Q8.1 Experiment - Privacy And Terms Coverage

## Experiment E8.2

Question: Is the current app legally launch-ready at the policy-link surface?

Method:

- Grep app source, docs, and tests for privacy/terms/legal strings in English and Vietnamese.
- Read actual route constants/router files instead of trusting prior docs.
- Read login, onboarding, settings, and data-control surfaces because the goal explicitly names those link locations.
- Record negative evidence separately from legal-content quality. This experiment checks existence and reachability, not legal sufficiency.

Commands:

```text
rg "privacy|terms|policy|tos|service|legal|điều khoản|quyền riêng tư|chính sách" lib web docs test -n
rg "Privacy|Terms|policy|privacy|terms|Điều khoản|Quyền riêng tư|Chính sách" lib\core\app_language.dart lib\features -n
Get-Content lib\app\navigation\app_route_constants.dart
Get-Content lib\app\navigation\routes\meta_routes.dart
Get-Content lib\features\auth\widgets\login_dialog.dart
Get-Content lib\features\onboarding\onboarding_screen.dart
Get-Content lib\features\me\me_screen.dart
Get-Content lib\features\me\screens\data_settings_screen.dart
```

Expected information gain: high. A route/link audit can falsify launch readiness quickly without drafting legal text.

## Experiment E8.3

Question: Can the missing legal route/link surface be cleared with a small tested slice while preserving review-needed legal status?

Method:

- Add route constants and GoRouter entries for `/privacy` and `/terms`.
- Add shared legal document and link widgets instead of duplicating route/link UI.
- Add VI + EN draft copy in `AppLanguage`, explicitly marked `review-needed draft`.
- Place links in Login, Onboarding, Me settings, and Data controls.
- Add focused route/widget tests, then fix only failures exposed by those tests.

Commands:

```text
flutter test test\app\navigation\app_route_smoke_test.dart --concurrency=1
flutter test test\features\auth\login_dialog_test.dart --concurrency=1
flutter test test\features\onboarding\onboarding_screen_test.dart --concurrency=1
flutter test test\features\me\data_settings_screen_test.dart --concurrency=1
```

Expected information gain: medium-high. This converts a route/link absence blocker into a legal-review blocker and catches regressions in the named surfaces.
