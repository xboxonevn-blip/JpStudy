# D8.Q8.3 Experiment - Firebase Auth Authorized Domains

## Experiment E8.5

Question: What can be verified from source/CLI about Firebase Auth authorized domains, and what must remain a manual launch gate?

Method:

- Read `lib/firebase_options.dart` for `authDomain`.
- Read `.firebaserc` and `firebase.json` for active Firebase project and Hosting targets.
- Run read-only Firebase Hosting site list to confirm live Hosting origins.
- Grep docs/source for `authorized domain`, `localhost`, `Google sign-in`, `OAuth`, and Firebase Auth domain language.
- Treat the Firebase Auth authorized-domain allowlist as unverified unless a source/CLI/API artifact exposes it directly.

Commands:

```text
rg "authorized domain|authorized domains|localhost|firebaseapp|web\\.app|Google sign-in|OAuth|authDomain" docs lib web firebase.json .firebaserc .github -n
npx firebase hosting:sites:list --project jpstudy-v2 --json
Get-Content lib\firebase_options.dart
Get-Content docs\FIREBASE_SECURITY_CHECKLIST.md
```

Expected information gain: medium. This cannot mutate the Console setting, but it can convert an implicit manual task into a concrete launch gate.
