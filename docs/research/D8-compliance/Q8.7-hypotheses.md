# D8.Q8.7 Hypotheses - Telemetry Retention and GDPR

## Question

Does the current telemetry design have enough consent, minimization, retention, and deletion controls for a closed beta?

## Hypotheses

- H8.7.1: Analytics collection is opt-in and respects browser Do Not Track.
- H8.7.2: Logged product events avoid obvious PII, free-text learner content, prompts, and answers.
- H8.7.3: Users can delete backup/study data, but cannot yet reset local Analytics identity from app UI.
- H8.7.4: GA4 retention is not source-controlled; final proof requires Console/Admin API access.
- H8.7.5: BigQuery export retention is not enforced by repo code; dataset/table TTL must be checked or set separately.
- H8.7.6: GDPR deletion is not fully operational until there is a runbook for Firebase Auth/Storage plus GA user deletion.
