# D5.Q5.5 Hypotheses - Mistake Review and Ghost Algorithm

## H5.5.1 - Mistake bank is a counter, not a separate scheduler

Prediction: `user_mistakes` stores `wrongCount` and `lastMistakeAt`, but no `nextReviewAt`, interval, ease, or FSRS state. Effect size: 1-3-7 due status is computed at read time from age buckets.

Falsifier: mistake rows have their own persisted schedule or use FSRS.

## H5.5.2 - Clearing requires multiple correct answers

Prediction: a new mistake starts at `MistakeDao.requiredCorrectStreak == 2`, `markCorrect()` decrements it, and the row is removed only after another correct answer when `wrongCount <= 1`.

Falsifier: one correct answer clears a new mistake, or clearing is based on accuracy/interval instead of a count.

## H5.5.3 - Grammar has two ghost concepts that can drift

Prediction: grammar uses `grammar_srs_state.ghostReviewsDue` for the current ghost provider, while older `LessonRepository.fetchGrammarGhosts()` / `markGrammarAsMastered()` uses incorrect `AttemptAnswer` rows. Effect size: at least one UI action can target the old ghost store and fail to clear the visible ghost.

Falsifier: all grammar ghost code reads/writes one shared source of truth.

## H5.5.4 - Ghost practice can feel productive without changing state

Prediction: `GhostPracticeScreen` score changes local UI only; answering ghost quiz questions does not call `GrammarRepository.recordReview()` or `MistakeRepository.markCorrect()`. Effect size: finishing ghost practice may leave `ghostReviewsDue` and mistake-bank rows unchanged unless another explicit action runs.

Falsifier: each ghost practice answer updates SRS and/or mistake state.

## H5.5.5 - Repeated failure can create a mistake loop

Prediction: duplicate wrong answers increment `wrongCount` indefinitely and due priority scales linearly by wrongCount, but there is no escalation path such as hint-only mode, prerequisite review, freeze, teacher note, or "too hard" branch.

Falsifier: high `wrongCount` triggers a different remediation strategy.
