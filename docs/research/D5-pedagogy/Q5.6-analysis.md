# D5.Q5.6 Analysis - Cross-Skill Prerequisites

## E5.6 Result

JP Study has useful building blocks for prerequisite remediation, but not enough coverage or routing to use it as a production learning policy. Cumulative kanji coverage improves the same-level graph, yet only `30.5%` to `39.1%` of kanji-bearing vocab entries are fully covered by known lower-or-same-level kanji assets.

## Findings

1. `H5.6.1` supported. Cumulative fully covered vocab is N1 `2483 / 6379`, N2 `1098 / 2991`, N3 `545 / 1786`, N4 `672 / 1719`, N5 `549 / 1470`. This is better than same-level coverage, but still too thin for gating.
2. `H5.6.2` supported. `KanjiPracticeArgs.kanjiIds` can open scoped read/write practice, and focused tests pass. No code path maps a weak vocab item to the missing kanji it requires.
3. `H5.6.3` supported. The only broad learning gate found is the foundations soft suggestion dialog, triggered once per surface when foundations progress is below `30%`.
4. `H5.6.4` supported. Mistake remediation records wrong items and can launch kanji practice for kanji mistakes. It does not escalate repeated vocab/grammar failure into prerequisite review.
5. `H5.6.5` supported. Grammar examples are well linked to grammar points, but grammar points do not declare required vocab, kanji, particles, or prior grammar.

## Production Impact

Hard prerequisite gating would block too many legitimate study paths and would be explainability-poor. For example, an N3 learner seeing a vocab item with kanji outside the local cumulative kanji set would not know whether the app is protecting them, hiding missing content, or simply failing to map available knowledge.

The good news is transport exists: scoped kanji practice can carry exact IDs, and vocab detail can already display kanji matches when they exist. The missing layer is a `PrerequisitePolicy` that is advisory first: detect missing kanji, show "study these first" suggestions, and measure whether learners recover faster. It should not lock content until coverage and telemetry prove the policy.

## Red Team

Steelman opposite: prerequisite gating could still help N5/N4 beginners because even partial coverage may catch the most common kanji. But the current numbers do not identify "common enough" or "already learned by this user"; they only show asset coverage. A hard gate would confuse content gaps with pedagogy.

## Decision

Do not ship hard cross-skill prerequisite gates. Ship advisory prerequisite suggestions only after adding item-level prerequisite extraction, coverage labels, and telemetry.

## Action

Next implementation should be `Q5.6b`:

1. Add a `PrerequisitePolicy` that extracts kanji dependencies from vocab terms and labels each dependency as learned, available-unlearned, or unavailable.
2. Add a vocab-detail advisory card: "Kanji to review first" with scoped practice via `KanjiPracticeArgs.kanjiIds`.
3. Add a high-mistake escalation branch for vocab mistakes: after repeated failure, suggest prerequisite kanji review before another quiz.
4. Track telemetry for suggestion shown, accepted, skipped, and downstream correctness.
5. Keep grammar prerequisite work deferred until grammar content declares explicit dependency metadata.
