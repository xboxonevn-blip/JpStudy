# D5.Q5.2 Hypotheses - Streak Rules

## H5.2.1 - Day boundary is device-local midnight, not a learner policy

Prediction: streak and daily-session logic use `DateTime.now()` and `DateTime(year, month, day)` directly, with no user timezone, server timezone, or configurable "next day starts at" setting. Effect size: every streak-relevant day key is derived from local clock only.

Falsifier: code stores a user timezone/day-boundary preference or uses a server-authoritative streak day.

## H5.2.2 - Streak has no freeze, repair, or subscription reinstatement path

Prediction: if the latest activity row is older than yesterday, `fetchProgressSummary()` reports streak `0`; no code path consumes a freeze, grants a grace day, or restores a streak after subscription/account state changes.

Falsifier: repository contains persisted streak freeze/repair inventory or a grace-period rule that preserves streak after a missed day.

## H5.2.3 - Global streak credit is not consistently awarded across skills

Prediction: vocab reviews and generic XP calls create/update `user_progress`, but grammar SRS review does not update global `user_progress`. Effect size: at least one learner-facing review path can complete study work without extending global streak.

Falsifier: all vocab, grammar, kanji, test, game, and daily-session completion paths call the same global activity recorder.

## H5.2.4 - UserProgress can duplicate a day under race or legacy data

Prediction: `user_progress.day` has no unique key, and `_ensureProgressRow()` uses SELECT-then-INSERT without transaction or upsert. Effect size: duplicate same-day rows are possible if two first activities race or old data is not normalized.

Falsifier: schema has a unique key on day and writes use an atomic insert-or-update.

## H5.2.5 - Tests cover presentation, not streak policy boundaries

Prediction: current tests cover milestones, home rendering, and week summaries, but not timezone/day-boundary, freeze/repair, cross-skill global credit, or duplicate-day behavior.

Falsifier: focused tests pin those policy cases.
