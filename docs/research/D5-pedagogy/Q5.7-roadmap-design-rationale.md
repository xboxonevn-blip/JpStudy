# D5.Q5.7 Roadmap Design Rationale - Textbook-Aligned Path

Date: 2026-05-15

## Question

Can the home learning path show a concrete per-level textbook sequence instead
of a generic daily plan, without pretending the local app has textbook coverage
it does not ship?

## Decision

Use a level-specific roadmap panel on the learning path screen. The panel is
advisory, not a hard gate. It maps each JLPT level to the source tracks already
present in the app:

- N5: Kana mastery, Minna I lessons 1-25, Hajimete N5, N5 mock review.
- N4: Minna II lessons 26-50, Hajimete N4, N4 mock and reading practice.
- N3/N2: Hajimete plus Shin Kanzen vocabulary, grammar, reading, listening,
  kanji, then mock/repair cycles.
- N1: Hajimete plus Shin Kanzen, mock/repair, then ongoing immersion through
  news, long reads, and manga-style reading.

## Why This Shape

The prior home path was useful for daily action, but it did not answer a
learner's source question: "Which book path am I actually following?" The new
panel keeps the action-first home design while adding a source-aware layer.

The important constraint is scope honesty. Local assets contain Minna only for
N5/N4; the official Minna series itself has intermediate books, but this app
does not currently ship that route. Therefore N3-N1 should show Hajimete and
Shin Kanzen rather than implying a Minna continuation.

## Verification

- `test/features/home/models/textbook_roadmap_test.dart` checks the per-level
  phase model for N5, N3, and N1.
- `test/features/home/learning_path_foundations_gate_test.dart` checks that the
  learning path renders the upper-level textbook roadmap for N3.

## Remaining Limits

The roadmap is not yet connected to per-textbook progress counters. It gives
orientation and source clarity first; progress normalization can come after the
lesson and review queues expose stable textbook-level completion stats.
