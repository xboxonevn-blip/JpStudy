# D5.Q5.1 Analysis - FSRS Correctness

## E5.1b Result

`FsrsService` now uses the current FSRS-6 default parameter vector (`21` values including decay), persists card `state` and `step`, and schedules new-card learning intervals at minute scale.

Pinned first-review intervals now match the local FSRS-6 product policy:

- `Again`: `1` minute.
- `Hard`: `5.5` minutes.
- `Good`: `10` minutes.
- `Easy`: `4` days.

## Findings

1. `H5.1.1` remediated. Local parameter count is now `21`, matching the current Open Spaced Repetition default vector.
2. `H5.1.2` remediated for new cards. The old `576` / `864` / `3,456` minute delays are gone.
3. `H5.1.3` remediated. Tests now pin exact learning-step intervals instead of only monotonic direction and clamps.
4. `H5.1.4` remains a calibration caveat. `0.9` retention is still a global default; no learner/cohort optimizer exists.
5. `H5.1.5` remediated. `srs_state`, `grammar_srs_state`, `kanji_srs_state`, and `kana_srs_state` now persist `fsrs_state` and `fsrs_step`; schema v30 migrates existing rows into Learning or Review state.
6. Dashboard/daily-plan critical counts now use FSRS learning/relearning state instead of the old `stability < 1.0` proxy.

## Production Impact

The P0 new-card delay is fixed. A learner pressing `Again` on a brand-new item now sees it after `1` minute, and `Good` stays in learning for `10` minutes instead of graduating days later. Review lapses now enter the `10` minute relearning step.

## Remaining Risk

This is now FSRS-6 scheduler correctness, not outcome calibration. The next evidence gap is whether the fixed scheduler improves real retention for Vietnamese JLPT learners once beta telemetry has enough SRS events.
