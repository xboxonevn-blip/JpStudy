# D5.Q5.3 Hypotheses - XP and Level Formula

## H5.3.1 - XP earning rules are fragmented by mode

Prediction: vocab SRS, learn mode, test mode, games, daily session, weekly challenge, and achievements each define their own XP values. Effect size: at least four independent XP formulas exist.

Falsifier: one shared XP policy service computes all rewards.

## H5.3.2 - Some visible XP does not reach global progress

Prediction: some screens display earned XP or achievement bonus XP without writing it to `user_progress.xp`. Effect size: learn-mode session XP and achievement `bonusXP` are persisted/displayed separately but not added to dashboard/global level.

Falsifier: every visible `+XP` value is routed through `recordStudyActivity()` or another global progress writer.

## H5.3.3 - There is no daily XP cap or burnout guard

Prediction: `recordStudyActivity()` and `recordReview()` add all XP deltas, weekly challenge bonus included, with no per-day max or diminishing returns.

Falsifier: code clamps daily XP or throttles rewards after a cap.

## H5.3.4 - Level curve is deterministic and covered, but comments are stale

Prediction: `LevelCalculator` uses an iterative increasing-gap curve (`100`, then +`150`, +`200`, ...), and tests pin the boundaries, but file comments still describe a quadratic progression that does not match the algorithm.

Falsifier: implementation and comments both use the same quadratic formula, or tests fail on current thresholds.

## H5.3.5 - XP formula is not pedagogically aligned to retention quality

Prediction: vocab SRS awards `Again=2`, `Hard=3`, `Good=5`, `Easy=7`, while wrong learn/test answers can earn zero and games can award score-based XP. There is no evidence that XP correlates with retention or mastery quality.

Falsifier: XP calculation incorporates retention, review debt, or calibrated learning outcomes.
