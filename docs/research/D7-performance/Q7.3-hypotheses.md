# D7.Q7.3 Hypotheses - Web Renderer Choice

## H7.3.1 - Current default build uses dart2js + CanvasKit

Prediction: the checked release build uses `compileTarget=dart2js` and `renderer=canvaskit`.

Falsifier: build config shows a Wasm/Skwasm primary target already.

## H7.3.2 - Wasm build succeeds but increases deploy artifact size

Prediction: `flutter build web --wasm` succeeds, but emits both Wasm/Skwasm and JS/CanvasKit fallback artifacts, increasing raw deploy bytes.

Falsifier: Wasm build fails, or total output is smaller than the default build.

## H7.3.3 - Wasm is not an immediate beta default without live compatibility checks

Prediction: official Flutter docs/build output still frame WebAssembly as a newer path, so beta should keep default CanvasKit until live Browser/Firebase Hosting checks prove Wasm safer.

Falsifier: official docs mark Wasm as the stable recommended default for all production web apps, and local/live smoke shows no compatibility or header risk.

## H7.3.4 - Resource-count smoke alone cannot decide renderer choice

Prediction: local Playwright resource count differs between default and Wasm, but browser/security-software/CDN behavior makes it insufficient without Lighthouse/trace/live probes.

Falsifier: resource/timing metrics are stable across repeated clean runs and clearly favor one renderer.
