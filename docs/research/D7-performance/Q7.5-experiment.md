# D7.Q7.5 Experiment - Asset Compression and Brotli

## Context

- Build: release `build/web` from the primary-only deploy at `2026-05-14T15:09:24Z`.
- Live target: `https://jpstudy.web.app`.
- Legacy target `https://jpstudy-v2.web.app` remains disabled and was not used for measurement.

## Method

1. Request representative live assets with `Accept-Encoding: br,gzip`, `gzip`, and `identity`.
2. Compare live `Content-Encoding`/`Content-Length` to local Node `zlib` gzip and Brotli sizes.
3. Recursively summarize local `build/web` raw, gzip, and Brotli size by file extension and largest files.

## Commands

```powershell
curl.exe -s -D - -o NUL -H "Accept-Encoding: br,gzip" "https://jpstudy.web.app/main.dart.js?d7q75=1"
curl.exe -s -D - -o NUL -H "Accept-Encoding: gzip" "https://jpstudy.web.app/main.dart.js?d7q75gzip=1"
curl.exe -s -D - -o NUL -H "Accept-Encoding: identity" "https://jpstudy.web.app/main.dart.js?d7q75raw=1"
curl.exe -s -D - -o NUL -H "Accept-Encoding: br,gzip" "https://jpstudy.web.app/canvaskit/canvaskit.wasm?d7q75=1"
curl.exe -s -D - -o NUL -H "Accept-Encoding: br,gzip" "https://jpstudy.web.app/assets/assets/data/content/grammar/n1/grammar_n1_1.json?d7q75=1"
curl.exe -s -D - -o NUL -H "Accept-Encoding: br,gzip" "https://jpstudy.web.app/assets/assets/images/mascot_fox_transparent.png?d7q75=1"
node -e "<zlib gzip/brotli size script>"
```

## Result

Firebase Hosting served Brotli for JS, wasm, JSON, HTML, and PNG requests when `br` was accepted. Local Brotli sizes matched live `Content-Length` closely.

Live representative headers:

| Asset | Encoding | Content-Length | Raw length |
|---|---:|---:|---:|
| `main.dart.js` | `br` | `1,306,529` | `6,279,333` |
| `main.dart.js` | `gzip` | `1,746,759` | `6,279,333` |
| `canvaskit/canvaskit.wasm` | `br` | `2,225,799` | `7,155,780` |
| `canvaskit/canvaskit.wasm` | `gzip` | `2,897,284` | `7,155,780` |
| `grammar_n1_1.json` | `br` | `1,765` | `17,793` |
| `mascot_fox_transparent.png` | `br` | `606,044` | `612,019` |
