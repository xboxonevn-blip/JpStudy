# D7.Q7.3 Experiment - Web Renderer Choice

## Experiment E7.5

Question: Should beta keep the default Flutter web renderer path or switch to `--wasm`/Skwasm?

Method:

- Read official Flutter web renderer/Wasm docs:
  - https://docs.flutter.dev/platform-integration/web/renderers
  - https://docs.flutter.dev/platform-integration/web/wasm
- Inspect default `build/web/flutter_bootstrap.js` build config.
- Build a separate Wasm output in `build/web_q73_wasm`.
- Compare default vs Wasm total bytes, top files, major payload families, and app JS/Wasm gzip sizes.
- Run the local resource smoke against default and Wasm outputs.
- Read `firebase.json` headers for current CSP/cross-origin posture.

Commands:

```text
node -e "parse _flutter.buildConfig from build/web/flutter_bootstrap.js"
flutter build web --release --wasm --base-href=/ --dart-define=JPSTUDY_RECAPTCHA_SITE_KEY=ci-placeholder --output build\web_q73_wasm
node -e "parse _flutter.buildConfig from build/web_q73_wasm/flutter_bootstrap.js"
Get-ChildItem -Recurse build\web_q73_wasm -File | Sort-Object Length -Descending | Select-Object -First 40
Compare total bytes for build\web and build\web_q73_wasm
Compute gzip sizes for main.dart.js, main.dart.wasm, main.dart.mjs
node tool\research\web_resource_smoke.js --build-root build\web --port 8112 --wait-ms 5000 --json
node tool\research\web_resource_smoke.js --build-root build\web_q73_wasm --port 8111 --wait-ms 5000 --json
Get-Content firebase.json
```
