# D7.Q7.6 Experiment - Font Subset Verification

## Context

- Build: current `build/web` from the Phase 8 primary-only deploy.
- Live target: `https://jpstudy.web.app`.
- Legacy `https://jpstudy-v2.web.app` stayed disabled and was not used as a performance target.

## Method

1. Read `build/web/assets/FontManifest.json`.
2. Measure local raw, gzip, and Brotli sizes for all bundled release font files.
3. Request the same font assets from primary Firebase Hosting with `Accept-Encoding: br,gzip`.
4. Run a live first-route resource smoke and inspect font fetches.

## Commands

```powershell
cmd /c type build\web\assets\FontManifest.json
Get-ChildItem -Recurse build\web -Include *.ttf,*.otf,*.woff,*.woff2
node -e "<fs + zlib font compression summary>"
curl.exe -s -D - -o NUL -H "Accept-Encoding: br,gzip" "https://jpstudy.web.app/assets/FontManifest.json?d7q76=1"
curl.exe -s -D - -o NUL -H "Accept-Encoding: br,gzip" "https://jpstudy.web.app/assets/fonts/MaterialIcons-Regular.otf?d7q76=1"
curl.exe -s -D - -o NUL -H "Accept-Encoding: br,gzip" "https://jpstudy.web.app/assets/assets/fonts/Manrope/Manrope%255Bwght%255D.ttf?d7q76=1"
curl.exe -s -D - -o NUL -H "Accept-Encoding: br,gzip" "https://jpstudy.web.app/assets/packages/cupertino_icons/assets/CupertinoIcons.ttf?d7q76=1"
node tool\research\web_resource_smoke.js --url https://jpstudy.web.app/?d7q76=1 --wait-ms 7000 --json
```

## Result

`FontManifest.json` contains three families:

- `MaterialIcons` -> `fonts/MaterialIcons-Regular.otf`
- `Manrope` -> `assets/fonts/Manrope/Manrope%5Bwght%5D.ttf`
- `packages/cupertino_icons/CupertinoIcons` -> `packages/cupertino_icons/assets/CupertinoIcons.ttf`

Local font sizes:

| Asset | Raw | gzip | Brotli |
|---|---:|---:|---:|
| `FontManifest.json` | `294` | `158` | `143` |
| `MaterialIcons-Regular.otf` | `51,140` | `28,469` | `24,852` |
| `Manrope[wght].ttf` | `165,420` | `68,306` | `59,306` |
| `CupertinoIcons.ttf` | `1,472` | `1,023` | `951` |

Live primary Hosting served each with `Content-Encoding: br`:

| Asset | Live `Content-Length` | Content type |
|---|---:|---|
| `FontManifest.json` | `145` | `application/json` |
| `MaterialIcons-Regular.otf` | `24,853` | `font/otf` |
| `Manrope[wght].ttf` | `59,307` | `font/ttf` |
| `CupertinoIcons.ttf` | `953` | `font/ttf` |

Live first-route smoke:

- `resourceCount`: `29`
- `jsonCount`: `1`
- `grammarResourceCount`: `0`
- FCP: `552 ms`

Font fetches observed in the first-route smoke:

| Resource | Encoded bytes | Decoded bytes |
|---|---:|---:|
| local `Manrope[wght].ttf` | `59,307` | `165,420` |
| remote `Roboto` woff2 | `63,464` | `63,464` |
| local `MaterialIcons-Regular.otf` | `24,853` | `51,140` |
| remote `Noto Sans SC` woff2 | `32,068` | `32,068` |
| remote `Noto Sans HK` woff2 | `18,596` | `18,596` |
| local `CupertinoIcons.ttf` | `953` | `1,472` |

## Notes

- Flutter's release build does subset icon fonts well.
- The remaining surprise is runtime fallback font fetching from `fonts.gstatic.com`, likely from Flutter web font fallback for CJK/system fallback coverage.
- The remote fallback fonts are smaller than the app shell but are still live first-route network requests and should be kept in the D7 live-resource budget.
