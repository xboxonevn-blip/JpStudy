# D7.Q7.2 Experiment - Bundle Composition

## Experiment E7.4

Question: What actually makes up the current Flutter web release bundle?

Method:

- Reuse the current release `build/web` from D7.Q7.1/Phase 1 verification.
- Group files by extension, top asset directory, and major payload family.
- List the largest release files and largest JSON files.
- Check for deferred/part JS chunks.
- Build a separate source-map copy in `build/web_q72_sourcemap` with CI placeholder App Check key, then inspect source-map source families.

Commands:

```text
Get-ChildItem -Path build\web -File | Select-Object Name,Length | Sort-Object Length -Descending
Get-ChildItem -Recurse build\web -File | Group-Object Extension | Sort-Object Count -Descending
Get-ChildItem -Recurse build\web\assets -File | group top asset path and sum bytes
Get-ChildItem -Recurse build\web -File | Sort-Object Length -Descending | Select-Object -First 40
Get-ChildItem -Recurse build\web -File | Where-Object { $_.Name -match 'part|deferred|main\.dart\.js_' }
flutter build web --release --base-href=/ --source-maps --dart-define=JPSTUDY_RECAPTCHA_SITE_KEY=ci-placeholder --output build\web_q72_sourcemap
node -e "parse build/web_q72_sourcemap/main.dart.js.map and group sources"
```
