# D7.Q7.5 Hypotheses - Asset Compression and Brotli

## Question

Is web performance still blocked by missing compression, or are the remaining transfer costs dominated by already-compressed runtime/assets?

## Hypotheses

- H7.5.1: Firebase Hosting serves Brotli for text and wasm assets when the browser advertises `Accept-Encoding: br,gzip`.
- H7.5.2: Brotli materially improves the app shell versus gzip, especially `main.dart.js`.
- H7.5.3: PNG/image assets gain little from HTTP Brotli and need image-specific optimization if they matter.
- H7.5.4: Pre-generating `.br`/`.gz` artifacts in the repo is unnecessary if Hosting already negotiates compression.
- H7.5.5: Remaining beta performance risk is resource shape and live timing, not a missing compression switch.
