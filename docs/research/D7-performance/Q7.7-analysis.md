# D7.Q7.7 Analysis - Kanji and Grammar Lazy-Load Verification

## Verdict

The lazy-loading picture is mixed.

Home is clean, and `/grammar` is correctly scoped to the active N4 level. But `/kanji` is not route-scoped: a direct Kanji route still pulls N4 grammar and N4 vocab JSON in addition to Kanji support JSON. This is better than the original all-level startup fetch, but it is not performance-ready for a route-specific live budget.

## Findings

1. Home no longer eagerly loads study content.
   - `/` fetched `27` resources.
   - It fetched `1` JSON resource, `FontManifest.json`.
   - It fetched `0` grammar, examples, vocab, or kanji content JSON.

2. Grammar route is active-level scoped.
   - `/grammar` fetched `25` N4 grammar JSON files and `25` N4 example JSON files.
   - It fetched no N5/N3/N2/N1 grammar.
   - It fetched no vocab or kanji JSON.

3. Kanji route still overfetches active-level study content.
   - `/kanji` fetched `126` resources and `101` JSON files.
   - It fetched `25` N4 grammar JSON, `25` N4 grammar-example JSON, and `46` N4 vocab JSON.
   - It fetched only `4` Kanji support JSON files, not level-specific kanji lesson JSON.

4. The overfetch is level-scoped, not all-level.
   - No N1/N2/N3/N5 grammar or vocab JSON was observed for N4.
   - That confirms the earlier D7 lazy-seeding fix reduced blast radius.

5. Home-only resource smoke is insufficient.
   - Home: `27` resources.
   - Grammar: `75` resources.
   - Kanji: `126` resources.
   - D7 needs route-specific live budgets for the shell routes, not just root.

## Hypothesis Results

- H7.7.1 supported: Home stays clean.
- H7.7.2 supported: Grammar route is level-scoped.
- H7.7.3 falsified: Kanji route is not route-scoped.
- H7.7.4 supported: extra content is active-level only.
- H7.7.5 supported: route-specific resource budgets are necessary.

## Product Impact

Closed beta can proceed if live first-route and user-perceived route timing are acceptable, but the Kanji route is not done from a performance perspective.

Recommended next D7 work:

1. Trace why Kanji direct route triggers active-level grammar and vocab seeding.
2. Add route-specific resource budgets for `/grammar`, `/kanji`, `/vocab`, and `/exam-center`.
3. Keep the D7 fix focused on route-bound seeding, not broad code splitting yet.
