# D7.Q7.6 Analysis - Font Subset Verification

## Verdict

Font subsetting is mostly healthy for beta. Flutter tree-shakes icon fonts well, Firebase Hosting Brotli-serves all bundled fonts, and local font transfer is small compared with `main.dart.js` and renderer/runtime assets.

The surprise is not local bundled fonts; it is live runtime fallback fetching from `fonts.gstatic.com`. First-route smoke pulled remote Roboto plus Noto Sans SC/HK subsets. These are not large enough to block beta, but they are external network requests and should stay visible in live resource budgets.

## Findings

1. Icon fonts are correctly reduced.
   - `MaterialIcons-Regular.otf` is `51,140` raw / `24,852` Brotli.
   - `CupertinoIcons.ttf` is `1,472` raw / `951` Brotli.
   - This supports the Flutter build log claim that icon tree-shaking is active.

2. Manrope is the largest bundled font but still modest.
   - `Manrope[wght].ttf` is `165,420` raw / `59,306` Brotli.
   - Live primary Hosting serves it as `font/ttf` with `Content-Encoding: br`.

3. Live first route still fetches remote fallback fonts.
   - Remote `Roboto`: `63,464` encoded bytes.
   - Remote `Noto Sans SC`: `32,068` encoded bytes.
   - Remote `Noto Sans HK`: `18,596` encoded bytes.
   - This likely comes from Flutter web font fallback handling for text/codepoint coverage, not from explicit app CSS imports.

4. Font transfer is not the dominant D7 problem.
   - Local bundled font transfer is about `85 KB` Brotli.
   - Observed live font transfer including remote fallback is about `200 KB` encoded.
   - `main.dart.js` alone transfers about `1.31 MB` Brotli and decodes to `6.29 MB`.

5. The resource budget should track external font fallback.
   - Current live first-route resource count is `29`.
   - Fonts are a meaningful subset of requests, but grammar/content eager loading remains fixed at `0` first-route grammar resources.

## Hypothesis Results

- H7.6.1 supported: Material Icons are below `100 KB` raw.
- H7.6.2 supported: Cupertino Icons are negligible.
- H7.6.3 supported: Manrope is the largest local font but below the predicted thresholds.
- H7.6.4 falsified: live first route fetched remote Roboto and Noto CJK fallback fonts.
- H7.6.5 supported: fonts are not the current performance blocker.

## Product Impact

Do not spend beta time on local icon/font subsetting. Keep the live resource smoke as the guard.

Later options:

1. Decide whether to self-host a deliberate Japanese/CJK subset or rely on Flutter's remote fallback.
2. Add live-route Lighthouse/trace before changing font strategy.
3. Keep `font-src` CSP scoped to self + `fonts.gstatic.com` while fallback fetching exists.
