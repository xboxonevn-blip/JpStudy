# D7.Q7.1 Experiment - Web Performance Baseline

## Experiment E7.1

Question: What is the current web release build/performance baseline while Lighthouse is unavailable?

Method:

- Run a release web build.
- Measure top build artifacts, total build size, asset size, JSON asset size, and gzip size for key launch files.
- Check for Lighthouse/browser CLI availability.
- Serve `build/web` locally and use Playwright performance APIs as a fallback smoke/timing probe.
- Record console/network failures separately from app load timing.

Commands:

```text
flutter build web --release --base-href=/
Get-ChildItem -Recurse build\web -File | Sort-Object Length -Descending | Select-Object -First 30 FullName,Length
Get-ChildItem build\web -File | Select-Object Name,Length
PowerShell gzip-size script for main.dart.js, canvaskit.wasm, sqlite3.wasm, drift_worker.js, flutter_bootstrap.js
if (Test-Path node_modules\.bin\lighthouse.cmd) { node_modules\.bin\lighthouse.cmd --version } else { 'lighthouse not installed' }
where.exe chrome; where.exe msedge; where.exe firebase; where.exe python; where.exe dart
python -m http.server 8097 --bind 127.0.0.1
Playwright navigate http://127.0.0.1:8097/
Playwright performance.getEntriesByType('navigation'/'paint'/'resource')
Playwright console/network summaries
```

Fixed local commit baseline: `e468d6c7ff3af9ebbe31abc5889efb7dfddab51a`

## Experiment E7.2

Question: Can D7 establish a reproducible build-artifact performance budget without Lighthouse/Chrome?

Method:

- Add `docs/research/D7-performance/web_perf_budget.json` with artifact-size budgets.
- Add `tool/research/web_perf_budget_report.dart` to scan `build/web`, compute raw/gzip bytes, list largest JSON files, and optionally fail on budget violations.
- Add CLI regression coverage in `test/tool/research/web_perf_budget_report_test.dart`.
- Run the exact report against current `build/web`.

Commands:

```text
flutter test test\tool\research\web_perf_budget_report_test.dart --concurrency=1
dart run tool\research\web_perf_budget_report.dart --build-root build\web --budget docs\research\D7-performance\web_perf_budget.json --fail-on-violation
```

Expected information gain: medium-high. This does not replace Lighthouse, but it creates a deterministic budget gate for the known structural risks: shell JS/wasm size, total build size, assets, and JSON payload.
