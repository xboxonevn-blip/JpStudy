# D7.Q7.7 Experiment - Kanji and Grammar Lazy-Load Verification

## Context

- Live target: `https://jpstudy.web.app`.
- Test state: fresh browser context, SharedPreferences values injected as JSON-encoded localStorage values.
- User state: onboarded N4, Vietnamese locale.
- Legacy `https://jpstudy-v2.web.app` stayed disabled and was not used.

## Method

For each route, open a fresh Playwright browser context, seed:

```js
localStorage.setItem('flutter.onboarding.completed', JSON.stringify(true));
localStorage.setItem('flutter.onboarding.level', JSON.stringify('n4'));
localStorage.setItem('flutter.app.locale', JSON.stringify('vi'));
```

Then navigate, wait 12 seconds, read `performance.getEntriesByType('resource')`, normalize same-origin paths, and count JSON groups.

Routes:

- `https://jpstudy.web.app/#/?q77=home`
- `https://jpstudy.web.app/#/grammar?q77=grammar`
- `https://jpstudy.web.app/#/kanji?q77=kanji`

## Commands

```powershell
node -e "<Playwright fresh-context resource counter for /, /grammar, /kanji>"
node -e "<Playwright per-level JSON counter for /grammar and /kanji>"
```

- Home remains clean for study-content JSON.
- Grammar direct route is level-scoped, not all-level.
