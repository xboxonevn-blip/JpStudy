# D2.Q2.1 Experiment - Vietnamese Content Review Status

## Experiment E2.1

Question: What is the current item-level Vietnamese review status by JLPT level and dataset?

Method:

- Compare current `assets/data/content` file count with the March 16 content audit scope.
- Add a reproducible structured scanner instead of grep-only counting:
  - `lib/core/research/content_vi_status_audit.dart`
  - `tool/research/content_vi_status_report.dart`
- Use TDD for the parser and CLI:
  - `test/core/research/content_vi_status_audit_test.dart`
  - `test/tool/research/content_vi_status_report_test.dart`
- Scan JSON only under `assets/data/content`.

Item extraction rules:

- `vocab` / `kanji`: each `entries[]` row is one item.
- `grammar`: each top-level grammar point row is one item.
- `grammar_examples`: each example row is one item; lower-level grouped examples inherit parent tags.
- `immersion`: each lesson JSON is one item.
- Files without a JLPT level are scanned but excluded from item-level JLPT summaries.

Signal rules:

- Machine-origin: `machine-translated-vi`
- Open review: `needs-vi-editorial` or `needs-human-review`
- Open review alias found during Q2.2 sampling and folded back into this scanner: `manual-review-needed`
- Approval: `vi-editorial-approved`, `human-reviewed`, `kanji-metadata-approved`, or `*Status: approved-by-user`
- Draft fields: tracked internally but not counted as open review without explicit review tags.

Commands:

```text
(rg --files assets\data\content | Measure-Object).Count
(rg --files -g '*.json' assets\data\content | Measure-Object).Count
flutter test test\core\research\content_vi_status_audit_test.dart test\tool\research\content_vi_status_report_test.dart
dart run tool\research\content_vi_status_report.dart --content-root assets\data\content
```
