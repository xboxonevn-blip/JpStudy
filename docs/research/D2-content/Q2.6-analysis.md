# D2.Q2.6 Analysis - Kanji Han-Viet / Unihan Spot Check

## E2.6 Result

The source trail exists, but the current N3/N2/N1 kanji metadata is not complete enough for learner-facing confidence. In a seeded 50-row sample, only `22 / 50` rows exactly matched Unihan `kVietnamese`; `28 / 50` had a mismatch or missing value.

## Hypothesis Updates

- H2.6.1 partially supported. Among rows with both local and Unihan values, `22 / 23` matched. The one sampled mismatch was `行`: local `Hành`, Unihan `hàng`.
- H2.6.2 supported. `23 / 50` sampled rows had missing local Han-Viet values. Some rows also lacked Unihan values, but the report classifies missing local first.
- H2.6.3 supported. `docs/credits/upper-jlpt-sources.md` credits Unicode Unihan `kVietnamese`, and the prior importer uses `.codex/sources/Unihan/Unihan_Readings.txt`; the credit trail does not imply row completeness.

## What This Means

- When local Han-Viet exists and Unihan has a value, the values are usually consistent.
- The larger learner risk is missing display metadata, especially blank/null Han-Viet fields across N3/N2/N1.
- Current tags such as `kanji-metadata-approved` are not enough for launch claims unless blank/null values are excluded or backfilled.
- Variant/traditional-Japanese characters can be missing from Unihan `kVietnamese`; they need a documented fallback or manual editorial decision.

## Red Team

This is a 50-row sample from 603 candidate rows, not a full corpus audit. The deterministic sample over-represents N3 blank values because later N3 lessons contain many empty `hanViet` fields. That is still relevant for learners because those rows are in the app path.

The `行` mismatch may be a legitimate multi-reading/Han-Viet ambiguity, not a simple error. It should be reviewed by a human editor rather than auto-replaced.

## Decision

Do not expose Han-Viet as a trusted upper-level learning aid until blank/null values are backfilled and ambiguous mismatches are reviewed. Use Unihan as a QA source, not as the only editorial authority.

## Action

1. Add a full-corpus upper-kanji completeness audit before editing values.
2. Backfill missing N3/N2/N1 Han-Viet values from Unihan where exact values exist.
3. Create a manual-review list for variant characters and multi-reading mismatches such as `行`.
4. Keep N3/N2/N1 kanji labels in beta wording as incomplete metadata until this pass is done.
