# D2.Q2.3 Analysis - Minna N3+ Coverage Gap

## E2.3 Result

Missing local Minna N3+ content is not a hard blocker for a 100-user beta if routes are labeled by JLPT level and not sold as a continuous Minna path. It is still a product-positioning gap because the official Minna series does continue into intermediate levels with Vietnamese support.

## Hypothesis Updates

- H2.3.1 supported. Vietnamese market/course evidence repeatedly maps Minna to N5/N4 and non-Minna series to N3+.
- H2.3.2 supported. Official 3A evidence shows Minna Chukyu I/II exists and has Vietnamese translation/grammar support, so "Minna stops at N4" is false for the textbook series.
- H2.3.3 supported. Local assets already provide N3-N1 vocab routes through `ShinKanzen` and `hajimete`; the safer product stance is JLPT-level coverage rather than Minna continuity.

## Red Team

The external market evidence is weak: course pages and cost tables are not learner surveys, and Google Trends comparison was not retrievable. This experiment distinguishes "obvious blocker" vs "not obvious blocker"; it does not prove Vietnamese N3+ learners prefer the current JpStudy route mix.

## Decision

Do not block beta solely on missing local Minna Chukyu. Do block any claim that the app offers a complete Minna path beyond N4.

## Action

1. UI/content copy should say `N3`, `N2`, `N1` or `ShinKanzen`/`hajimete`, not "Minna continuation."
2. Add Minna Chukyu as a future content candidate, not launch-critical scope.
3. Use Q2.5 scope comparison to decide whether N3-N1 counts are sufficient vs official JLPT rough scope.
4. Real beta onboarding should ask which textbook path users come from; this is better evidence than web-course pages.
