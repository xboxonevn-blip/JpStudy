# D2.Q2.4 Analysis - Cross-Link Coverage Graph

## E2.4 Result

Grammar examples are strongly linked to grammar points, but vocab to kanji prerequisite coverage is shallow. The app has enough data to show grammar examples next to grammar points, but not enough same-level kanji coverage to safely infer "learn these kanji before this vocab."

## Hypothesis Updates

- H2.4.1 supported. Matched grammar example groups: N1 `976 / 980`, N2 `764 / 764`, N3 `100 / 100`, N4 `100 / 100`, N5 `118 / 118`.
- H2.4.2 supported. Fully covered kanji-bearing vocab entries are low: N1 `780 / 6,379`, N2 `504 / 2,991`, N3 `188 / 1,786`, N4 `393 / 1,719`, N5 `549 / 1,470`.
- H2.4.3 supported. Link schema differs: N1/N2 kanji examples use word links, N3-N5 use `sourceVocabId` refs. N4/N5 refs are not fully resolved.

  - N2 `199 / 1,097`
  - N3 `173 / 1,099`
  - N4 `137 / 857`

- N3 source refs resolve `155 / 155` after matching against `lemma.vocabId`.

## Red Team

Same-level coverage is a harsh metric because higher-level vocab can include lower-level kanji that should not be duplicated in the same level. A better next graph should use cumulative kanji known by or before a level. Even with that caveat, the current graph shows explicit prerequisite modeling is not ready.

## Decision

Do not add prerequisite gating yet. Add graph diagnostics first, then move to cumulative coverage and missing-link samples.
