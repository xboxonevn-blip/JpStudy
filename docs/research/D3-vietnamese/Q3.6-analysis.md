# D3.Q3.6 Analysis - Plural and ICU Risk

## Result

- `Intl.plural` / `plural` API usage: `0` matches.
- Manual singular/plural guards: `5` relevant user-facing matches.
- Raw English count-plural string matches: `41` across `18` files.
- Interpolated string surface in `lib/`: `6,942` matches.

## Hypothesis Updates

- H3.6.1 supported. There is no ICU plural machinery in the current source.
- H3.6.2 supported. The grep found `41` raw English count strings with plural nouns.
- H3.6.3 supported. Only `5` relevant user-facing strings guard singular/plural explicitly.
- H3.6.4 supported. Vietnamese copy is usually safe with `$count mục` / `$count câu`; English labels can silently show `1 items`, `1 terms`, `1 questions`, `1 lessons`, or `1 strokes`.

## Interpretation

This is a real English/Japanese polish issue, but not a reason to do full ARB migration before beta. The highest-risk fixes can be handled inside the current architecture by adding count-label helpers or small English singular guards in feature-copy modules.

The core problem is duplicated count wording. `app_language.dart` already has count helpers such as `itemsCountLabel`, `lessonCountLabel`, `unansweredSubmitLabel`, and `termsNeedPracticeLabel`; these should become the central place for singular-aware English, while Vietnamese and Japanese can keep count-neutral forms.

## Recommendation

Do not start ARB migration for Q3.6 alone. Add a small plural helper in `AppLanguage` or feature-copy modules:

```dart
String enCount(int count, String singular, String plural) =>
    '$count ${count == 1 ? singular : plural}';
```

Then patch high-frequency English strings with tests. If a future ARB pilot happens, these helper call sites are the safest initial candidates for ICU migration.

## Red Team

- This regex pass misses plural bugs in strings that use words outside the searched noun list.
- It may include strings rarely seen by Vietnamese beta learners, so launch impact is lower than content quality and critical path bugs.
- Japanese count grammar is also not fully audited; Q3.6 mainly proves English plural risk.
- ARB/ICU would solve this class more cleanly long-term, but the measured surface still favors staged fixes.
