# D3.Q3.2 Hypotheses - Hardcoded Vietnamese Outside AppLanguage

## H3.2.1 - AppLanguage is not the only UI-copy source

Prediction: `lib/` contains many hardcoded Vietnamese strings outside `lib/core/app_language.dart`, especially in feature screens using `AppLanguage.vi =>` switches or local `_tr` helpers.

Falsifier: fewer than 20 non-`app_language.dart` Dart lines contain Vietnamese diacritics.

## H3.2.2 - Most hardcoded strings are localized switches, not accidental literals

Prediction: many hits are legitimate local switch/helper patterns, but they still bypass the central language file and make editorial QA harder.

Falsifier: more than 80% of hits are comments or non-UI data constants.

## H3.2.3 - Hardcoded Vietnamese concentrates in a few modules

Prediction: the top five files by hardcoded Vietnamese hits account for most instances.

Falsifier: hits are evenly spread across more than 50 files.
