# D3.Q3.5 Experiment - ARB Migration Recommendation

## Cheapest Experiment

Measure actual migration surface from code instead of trusting the prompt count, then compare effort against D3.Q3.1-Q3.4 risks.

## Commands

```text
rg -n "$" lib\core\app_language.dart | Select-Object -Last 1
dart run tool\research\vietnamese_i18n_audit_report.dart --app-language lib\core\app_language.dart --lib-root lib --content-root assets\data\content --docs-root docs
(rg -l "AppLanguage" lib | Measure-Object).Count
(rg -l "appLanguageProvider" lib | Measure-Object).Count
(rg -o "AppLanguage\.(en|vi|ja)" lib | Measure-Object).Count
(rg -n "case AppLanguage\.(en|vi|ja)|AppLanguage\.(en|vi|ja) =>" lib | Measure-Object).Count
if (Test-Path l10n.yaml) { Get-Content l10n.yaml } else { 'NO l10n.yaml' }
(Get-ChildItem -Recurse -Filter *.arb | Measure-Object).Count
Get-Content pubspec.yaml
Get-Content lib\app\app.dart
Get-Content lib\core\language_provider.dart
```

## Decision Criteria

Recommend full ARB migration before beta only if:

1. Call-site rewrite surface is small enough to finish and QA in under `2` days.
2. Current risks are mostly missing ICU/plurals, not wording/glossary/ownership.
3. Migration does not force providers/models/export services to depend awkwardly on `BuildContext`.

Otherwise recommend staged migration.
