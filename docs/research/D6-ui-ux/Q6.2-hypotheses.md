# D6.Q6.2 Hypotheses - Empty State Audit

## H6.2.1 - Empty states are locally implemented

Prediction: most feature modules implement empty states in screen-local widgets instead of a shared `EmptyState` primitive. Effect size: fewer than 30% of sampled empty states use one shared component.

Falsifier: a shared empty/error state widget covers most sampled modules.

## H6.2.2 - CTA quality varies by module

Prediction: some empty states have a recovery CTA, while others only show text or disappear with `SizedBox.shrink()`. Effect size: at least 5 sampled states lack a clear next action.

Falsifier: nearly every empty state includes a visible CTA to populate, retry, browse, or change filters.

## H6.2.3 - Vietnamese localization is uneven

Prediction: user-facing empty-state copy is mixed between `AppLanguage` helpers and hardcoded local strings. Effect size: at least 5 sampled states use local hardcoded copy.

Falsifier: sampled states route all copy through `AppLanguage` or feature copy helpers.

## H6.2.4 - Visual richness is inconsistent

Prediction: empty states range from polished cards with icons to plain centered text or hidden widgets. Effect size: at least 4 visual styles in the sample.

Falsifier: empty states share a common card/icon/caption structure.

## H6.2.5 - Silent empty states hide important product gaps

Prediction: some modules suppress empty/error/loading content entirely, which makes missing data look like absent product surface rather than a recoverable state.

Falsifier: no sampled module uses `SizedBox.shrink()` for empty/error states in a primary user flow.
