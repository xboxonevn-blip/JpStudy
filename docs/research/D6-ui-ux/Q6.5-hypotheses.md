# D6.Q6.5 Hypotheses - Contrast Audit

## H6.5.1 - Core text tokens pass, muted helper tokens are the risk

Prediction: primary/body text passes WCAG AA, while helper/caption/hint alpha tokens near `0.42`-`0.55` fail or only pass for large text.

Falsifier: all sampled muted text tokens meet `4.5:1` for normal text.

## H6.5.2 - Navigation contrast is acceptable

Prediction: sidebar and mobile nav inactive labels use enough opacity (`0.64`-`0.72`) to pass normal-text contrast.

Falsifier: inactive nav labels fall below `4.5:1`.

## H6.5.3 - Status/accent colors fail when used as small text

Prediction: warning/success/info/accent colors are fine for icon/large display accents but weak as small text on light surfaces.

Falsifier: semantic color text on light surfaces consistently reaches `4.5:1`.

## H6.5.4 - Dark mode token contrast is better than light muted contrast

Prediction: dark base/body colors pass more often because `ink` is very light on dark surfaces.

Falsifier: dark muted tokens are the primary contrast risk.

## H6.5.5 - Disabled controls are exempt but still visually weak

Prediction: disabled button labels and disabled-ish alpha tokens fail `4.5:1`, but WCAG exempts inactive controls. Product risk remains discoverability, not formal AA failure.

Falsifier: disabled controls are both exempt and visually clear enough under sampled colors.
