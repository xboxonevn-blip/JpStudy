## Caveats

- Static Dart code cannot prove final Flutter render-box or semantic hit region where ancestor padding changes the tappable area.
- Desktop-only click targets may be lower-severity than mobile-first controls, but the app is a mobile learning product, so shared widgets should keep a `44x44` floor.
- Chip/filter controls need a widget-test or browser hitbox pass because Material defaults can be altered by density/theme/ancestor layout.
