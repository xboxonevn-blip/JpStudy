# D6.Q6.5 Analysis - Contrast Audit

## E6.5 Result

Core navigation and primary text tokens are acceptable, but small muted/helper text and semantic color chips do not consistently meet WCAG 2.1 AA for normal text. The worst measured token is input hint text at `2.57:1`; warning-colored small text is also below normal-text AA.

- Light semantic foregrounds were darkened in `AppThemePalette.light`: success `#176C4A`, warning `#7A4D00`, error `#9B3440`, info `#315F9A`, accent `#A64926`.
- `InputDecorationTheme.hintStyle` was raised from `ink 0.42` to `ink 0.68`.
- `test/app/theme/app_theme_language_test.dart` now guards light semantic foregrounds, input hints, and warning status chips at `>=4.5:1`.

## Findings

1. `H6.5.1` supported. `bodyMedium` and nav labels pass, but `ink` alpha values `0.42`, `0.45`, `0.50`, and `0.55` fail or only pass for large text on light surfaces.
2. `H6.5.2` supported. Sidebar inactive labels (`6.35:1`-`6.43:1`) and mobile nav inactive labels (`4.97:1`) meet normal-text AA.
3. `H6.5.3` supported. Light semantic colors are weak as small text: warning `2.98:1`, success `4.00:1`, info `4.40:1`, error `4.27:1`, accent `3.29:1`. `AppStatusChip.warning` is only `2.55:1`.
4. `H6.5.4` mostly supported. Dark `bodyMedium`, `primary`, and `ink 0.55` pass; dark `ink 0.45` is large-only at `3.99:1`.
5. `H6.5.5` supported. Disabled-ish `ink 0.35` is `2.14:1`; inactive controls are exempt, but the same alpha should not be reused for active helper text.

## Production Impact

This is a beta accessibility blocker for small text, not for the main reading path. The app can be readable overall while still failing AA in helper text, hint text, status chips, and small semantic labels. Vietnamese learners with older devices, sunlight glare, or low vision will lose exactly the context that explains what to do next.

The highest-risk items are:

1. Input hint text at `ink 0.42`.
2. Small `warning` labels/chips and `AppStatusChip.warning`.
3. Small semantic chips using `success`, `info`, `error`, or `accent` as text on light surfaces.
4. Empty-state/helper captions using `ink 0.55` at `11`-`13px`.

## Red Team

Steelman opposite: many sampled failures are helper text or inactive controls, not primary content. Some chips use icon/shape/color redundantly, and large text only needs `3:1`. That lowers severity, but does not clear beta AA because several uses are normal-size active labels (`11`-`13px`) and hints.

## Decision

Do not attempt a full visual redesign. Define contrast-safe token rules:

1. Normal active helper text on light surfaces should use at least `ink 0.62` (`4.56:1`) or stronger.
2. Placeholder/hint text should not use `ink 0.42`; raise to at least `ink 0.62` or make it non-essential and add visible labels.
3. Semantic colors should not be used directly for small text on light surfaces unless darkened or paired with a contrast-safe foreground.
4. `AppStatusChip.warning/success` should use a darker foreground token or `palette.ink` text with colored icon/border.
5. Disabled controls may remain lower contrast, but disabled explanatory copy must use active text contrast.

## Action

are complete. Remaining follow-up:

1. Convert small ad hoc semantic label/chip components (`_GradePill`, `_InfoBadge`, `_RatingButton` or their shared equivalents) only where they bypass the shared palette-safe tokens.
