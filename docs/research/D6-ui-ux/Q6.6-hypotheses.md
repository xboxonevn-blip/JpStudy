# D6.Q6.6 Hypotheses - Touch Target Audit

## H6.6.1 - Primary navigation and practice CTAs pass

Prediction: bottom navigation, sidebar rows, main practice buttons, and quiz advance/rating controls meet the `44x44` touch target floor.

Falsifier: any primary route or quiz progression control is intentionally constrained below `44x44`.

## H6.6.2 - Default Material buttons/chips are mostly safe

Prediction: controls that rely on normal Material `ButtonStyleButton`, `IconButton`, `ChoiceChip`, and `FilterChip` defaults are safer than custom `GestureDetector`/`InkWell` wrappers.

Falsifier: the app theme globally shrinks Material tap targets or most sampled default controls are below `44x44`.

## H6.6.3 - Dense/compact overrides are the main risk

Prediction: explicit `shrinkWrap`, zero constraints, compact visual density, or `minimumSize` overrides create the real failures.

Falsifier: no active controls with these overrides are under `44x44`.

## H6.6.4 - Kanji/radical grids pass by cell geometry

Prediction: Kanji Hub tiles and radical tiles pass because grid cells are `70` and `88` max extents, even when internal text is small.

Falsifier: the interactive grid cells are visually or hitbox-constrained below `44x44`.

## H6.6.5 - Custom rating/tiny action controls need policy

Prediction: custom star ratings, inline lesson actions, and small text CTAs lack a shared touch-target policy.

Falsifier: every custom interactive wrapper adds an explicit `44x44` min hitbox.
