# D6.Q6.5 Experiment - Contrast Audit

## Experiment E6.5

Question: Do sampled navigation, hint, caption, status, and disabled-style colors meet WCAG 2.1 AA contrast expectations?

Method:

- Read theme palette and shared component code directly.
- Search for muted alpha text, disabled controls, hint text, status colors, and caption/subtitle patterns.
- Compute WCAG contrast ratios for sampled foreground/background pairs by alpha-blending translucent Flutter colors over their actual light/dark surfaces.
- Classify normal text with `4.5:1`, large text with `3:1`, and inactive/disabled UI as exempt but risky when content-like.

Commands:

```text
Get-Content -Path lib\app\theme\app_theme_palette.dart
Get-Content -Path lib\app\theme\app_theme.dart
Get-Content -Path lib\app\navigation\app_shell_scaffold.dart
rg -n "withValues\(alpha: 0\.[0-9]+\)|withOpacity\(0\.[0-9]+\)|disabled|hint|tagline|inactive|secondary|outline|caption|bodySmall|labelSmall" lib\app lib\core lib\features -g "*.dart"
```
