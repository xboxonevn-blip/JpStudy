# D6.Q6.6 Experiment - Touch Target Audit

## Experiment E6.6

Question: Do sampled interactive controls meet the WCAG/mobile convention of at least `44x44` logical pixels?

Method:

- Read navigation, top-bar, Kanji Hub, review, handwriting, lesson, library, practice, and mistake-list code directly.
- Grep for explicit minimum-size overrides, shrink-wrapped tap targets, compact constraints, custom `GestureDetector`/`InkWell` controls, and grid delegates.
- Classify controls as pass/risk/fail from static geometry. Default Material controls were treated as likely pass unless local code overrides constraints/tap target.
- No runtime semantics/hitbox snapshot was taken, so this is a static audit, not a final accessibility-tree measurement.

Commands:

```text
rg -n -C 6 "minimumSize: Size|tapTargetSize: MaterialTapTargetSize\.shrinkWrap|Size\.fromHeight|Size\.fromWidth|fixedSize:|constraints: BoxConstraints|height: 44|width: 44|size: 44|BoxConstraints\(" lib -g "*.dart"
```
