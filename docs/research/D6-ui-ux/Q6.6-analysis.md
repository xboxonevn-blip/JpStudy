# D6.Q6.6 Analysis - Touch Target Audit

## E6.6 Result

Primary navigation, quiz progression, and Kanji Hub grids mostly meet the `44x44` touch target floor. The beta risk is concentrated in compact/dense overrides: small icon buttons, shrink-wrapped text CTAs, and custom `GestureDetector` wrappers.

- Added `AppTouchTargets.min = 44` in `app_spacing.dart`.
- Raised compact notification, Discover reorder, Discover focus chip, mistake delete, and interactive `StarRating` targets to at least `44x44`.
- Added focused widget coverage for compact top bar, dense Discover controls, mistake delete, and interactive star targets.

Remaining touch-target work is the lower-risk route-specific sweep: lesson inline icon row, library/practice compact action CTAs, handwriting free-practice CTA, and any remaining zero-constraint active controls.

## Findings

1. `H6.6.1` mostly supported. Mobile bottom nav is `82` high, sidebar rows are large, Kanji quiz ratings are `84x44`, and the quiz next button is `160x48`.
2. `H6.6.2` supported with caveat. No global chip/button tap-target override was found in the theme; default Material controls are likely safer than custom wrappers. Runtime widget tests should confirm chip render sizes.
3. `H6.6.3` strongly supported. Every clear failure uses explicit compacting: `shrinkWrap`, `minimumSize: Size(0, 0)`, `BoxConstraints.tightFor(36/40)`, `BoxConstraints()`, or a custom `GestureDetector`.
4. `H6.6.4` supported. Kanji and radical grids use cells at `70` and `88` max extents, so the interactive tile area is not the blocker.
5. `H6.6.5` supported. `StarRating`, `_FocusChip`, inline lesson icons, and compact action links have no shared min-hitbox policy.

## Production Impact

This is a mobile/tablet accessibility blocker for secondary controls, not a blocker for starting core practice. A learner can navigate and answer main quiz prompts, but small controls for reordering, filtering, deleting mistakes, starring/marking lessons, opening resource links, and switching to free handwriting practice are precision targets.

The most learner-visible risks are:

1. Discover Practice focus/reorder controls.
2. Top-bar notification icon on compact widths.
3. Lesson detail learned/star/edit controls.
4. Mistake delete buttons.
5. Library/practice small action CTAs.

## Red Team

Steelman opposite: many failures are secondary or desktop-style controls, and some are text links where users can still click accurately with a mouse. That lowers urgency relative to primary quiz/reading CTAs. It does not clear the issue for beta because the same widgets appear in mobile-width or learner-critical surfaces, and touch precision is worse for older/tablet users from D4.P4.

## Decision

Do not redesign the surfaces. Add a small touch-target policy:

1. Active controls must expose at least `44x44` hit area, even when the visual affordance is smaller.
2. Ban `tapTargetSize: MaterialTapTargetSize.shrinkWrap`, zero constraints, and `minimumSize: Size(0, 0)` for active mobile controls unless wrapped in a `44x44` hitbox.
3. Add a shared helper/token such as `AppTouchTargets.min = 44` or a `MinTouchTarget` widget.
4. Fix the explicit violations first: discover panel controls, global top bar notification, lesson inline icon row, mistake delete, library/practice action links, handwriting free-practice CTA, and `StarRating`.
5. Add widget tests that assert rendered control boxes are at least `44` in the highest-risk compact modes.
