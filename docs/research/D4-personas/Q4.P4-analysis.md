# D4.Q4.P4 Analysis - Bác Hùng N4

Commit: `e468d6c7`
Date: 2026-05-14

## Result

P4 is not production-ready for an older tablet-only learner. Root and the post-tap `Học` plan can preserve N4 and are readable at 125% zoom, but direct study routes on live still fall to N5, there is no first-class travel/fun goal, and no visible font-size/accessibility setting.

## Hypothesis Updates

H4.P4.1 - N4 trust: falsified on live. Root shows N4 and the slow-tap learning path keeps N4, but direct kanji, vocab, grammar, reading, and study hub fall to N5.

H4.P4.2 - Large-font tablet comfort: partially supported. At `834x1194` with 125% zoom, major text and primary buttons remain readable. However, the app collapses to a phone-style stacked shell, hides dense plan context, and direct-route modals obscure content.

H4.P4.3 - Slow-tap forgiveness: partially supported. A slow tap on the root plan path opens a predictable N4 `Học` surface with large primary buttons. The root compact cards use text-sized CTAs, and there is no explicit slow-tap/accessibility mode.

H4.P4.4 - Fun/travel motivation: weakly supported. `reading` is the closest available goal; `travel` does not exist. The N4 learning plan exposes `Mở immersion`, but exam prep and JLPT labels remain dominant.

H4.P4.5 - N4 content confidence: falsified by entry path. Root path shows N4 lesson 26 and N4 plan steps, but direct core content routes show N5 and zero/low-confidence states.

## Severity

- HIGH: Live direct routes still drop N4 users to N5.
- HIGH: No travel/fun study goal; retiree-tour motivation must squeeze into reading/JLPT.
- MEDIUM: Tablet at 125% uses mobile shell and loses detailed plan density.
- MEDIUM: No visible font-size/accessibility preference beyond browser/OS zoom.
- MEDIUM: Foundations modal blocks first access on direct N4 routes and compounds the N5 fallback.

## What Worked

- Root N4/VI state is visually clean at tablet portrait.
- Slow tap successfully opens a readable N4 plan screen.
- The post-tap plan gives a concrete `~15p` lesson path and an `Mở immersion` option.
- Bottom nav targets are large and readable at 125%.
- Community/settings surface includes data/reset and UI lab shortcuts.

## Red Team

The large-font proxy used browser zoom, not an OS text-scale signal. Flutter web may respond differently to real tablet accessibility text settings. The root `Học` path was tested by coordinate tap because Flutter semantics are sparse in Playwright snapshots; visual evidence supports the outcome, but exact target hitbox measurement needs widget/browser instrumentation.

The direct-route failures are likely stale deploy drift, not new local source defects. They still count as production evidence because the live site is the beta candidate.

## Action

1. Redeploy or verify the hosting channel, then rerun P2-P4 direct routes as one matrix.
2. Add a beta-scope decision for `StudyGoal`: if retirees/tour learners are target users, add travel/fun/culture onboarding or avoid that positioning.
3. Add a tablet/large-font smoke test around root, learning plan, and bottom nav.
4. Consider a visible accessibility setting or onboarding hint for large text/reduced motion.
5. Re-test P4 after channel parity before patching per-screen N4 content.

## Mental Model

Bác Hùng không cần nhiều tính năng hơn; ông cần chữ to, đường đi ít nhầm, và mục tiêu hợp đời sống. Nếu app tự rơi về N5 hoặc chỉ nói chuyện thi JLPT, người học vì du lịch sẽ thấy app không hiểu mình dù nội dung N4 có tồn tại.
