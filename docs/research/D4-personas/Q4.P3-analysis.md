# D4.Q4.P3 Analysis - Mai N2

Commit: `e468d6c7`
Date: 2026-05-14

## Result

P3 is not production-ready for a laptop-heavy N2 crammer. Root startup can show N2 in Vietnamese, but direct study routes on live still collapse to N5, exam entry is stale/empty, and there is no visible 3-hour cramming mode.

## Hypothesis Updates

H4.P3.1 - Level persistence: falsified on live. Root cold start shows N2, but direct grammar, vocab, kanji, coach, and reading hard loads show N5. Local code already has an app-shell init patch and test, so this is currently a live-channel parity blocker.

H4.P3.2 - Cram planning: falsified. The clean N2 root plan is `~14 phút`, coach falls to N5 on direct load, and no 3-hour/day or July-JLPT cramming path is visible.

H4.P3.3 - N2 content confidence: falsified by entry path. N2 content completeness cannot be trusted because primary direct routes show lower-level content; exam center shows a blank legacy chooser.

H4.P3.4 - Group study: partially supported. Leaderboard has `Chia sẻ snapshot` and a `Bạn bè` scope; community shows friend/community concepts, but invite and room features are `Sớm`, `Tiếp theo`, or `Kế hoạch`.

H4.P3.5 - Laptop throughput: partially supported. The 1366x768 layout is readable and sidebar navigation is fast, but foundations modals and level fallback break the rapid cramming loop.

## Severity

- HIGH: Live direct routes ignore N2 persisted state and silently show N5 study lanes.
- HIGH: Live `/exam-center` remains stale/empty against local richer route expectations.
- HIGH: No cramming/intensity model; a 3-hour/day N2 learner receives a short default plan.
- MEDIUM: N2 social loop is mostly roadmap, not an active group-study feature.
- MEDIUM: Foundations modal interrupts direct study routes; on live P3 this compounds the N5 fallback.

## What Worked

- Root start honors JSON-encoded N2/VI prefs.
- Desktop shell, sidebar, and Vietnamese typography are readable at `1366x768`.
- Leaderboard has a concrete share action.
- Community screen exposes future invite/community-room direction.
- Reading route desktop CTA is visible in the first viewport.

## Red Team

The live direct-route failures likely repeat stale deploy drift already found in P2. That lowers confidence in assigning those failures to current local source, but it raises production risk: beta users only experience the deployed channel.

The P3 daily-time hypothesis used persona context, not a seeded app preference, because the current onboarding model stores only level and goal. This is still product evidence: the app has no first-class way to represent Mai's 3-hour cramming constraint.

## Action

1. Redeploy or verify the `jpstudy-v2` hosting channel, then rerun P2/P3 direct-route probes.
2. Add a beta-scope decision for cramming intensity: either build a long-session JLPT plan or exclude 3-hour/day crammers from the target persona set.
3. If cramming stays in scope, add an N2 coach/exam path that exposes full mock, repair drills, reading, vocab, and grammar in one desktop workflow.
4. Clarify social scope: ship share-only as beta, or implement invite/group-room affordances before positioning for group study.
5. Re-test live `/exam-center` after deploy/channel fix before spending more local time on exam routing.

## Mental Model

Mai does not need a gentle daily habit app; she needs a trustworthy N2 command center for long study blocks. If a direct link or exam button drops her to N5, the app stops being a study partner and becomes another thing she must debug before studying.
