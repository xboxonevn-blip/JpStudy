# D8.Q8.1 Experiment - Release Risk Inventory

## Experiment E8.1

Question: What release risks remain after D1 BigQuery identity unblock, D6 UI audits, and D7 web build baseline?

Method:

- Inspect Firebase Hosting config, targets, live channel metadata, shipping docs, security checklist, web shell, Firebase init, and App Check activation.
- Run focused release smoke tests for shell layout and key app routes.
- Use read-only Firebase CLI commands only; do not deploy.
- Fold known D1/D6/D7 blockers into a single beta-release risk stack.

Commands:

```text
flutter test test\app\layout\responsive_shell_test.dart --concurrency=1
flutter test test\app\navigation\app_route_smoke_test.dart --concurrency=1
flutter test test\features\ui\exam_and_coach_route_smoke_test.dart --concurrency=1
npx firebase-tools hosting:sites:list --project jpstudy-v2 --json
npx firebase-tools hosting:channel:list --site jpstudy-v2 --project jpstudy-v2 --json
npx firebase-tools hosting:channel:list --site jpstudy --project jpstudy-v2 --json
```

Expected information gain: high. This does not ship a release, but it identifies whether beta blockers are deploy hygiene, test drift, data-table availability, or product readiness.
