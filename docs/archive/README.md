# Docs Archive

`docs/archive/` stores historical documentation that is no longer part of the main active docs flow.

Use this area for:
- outdated implementation plans
- walkthroughs for completed legacy features
- historical references worth keeping but not worth surfacing prominently

Keep active planning in `docs/plans/` and generated outputs in `docs/reports/`.

## Retired local-only frameworks

Some historical local agent/tooling setups may be removed from the repo to keep the project portable. If preserved, they should live outside the repository.
