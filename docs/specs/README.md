# Specs

`docs/specs/` contains feature or product specifications.

Use this folder for documents that define a target behavior, experience, or requirement set more formally than a note or plan.

## Difference from other folders

- `docs/specs/`: what the feature should be
- `docs/plans/`: how to implement it
- `docs/notes/`: exploration, rough thinking, references
- `docs/reports/`: outputs after analysis or execution
